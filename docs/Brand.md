## Visual Direction

**Reference:** Linear's clean structure with a hint of Notion's warmth. Dark-first, minimal, modern. Thin borders, generous spacing, no visual clutter.

There is no light mode. The UI lives on dark backgrounds.

---

## Color Palette

### Foundation

| Role | Color | Hex | Usage |
|------|-------|-----|-------|
| **Background** | Near-black | `#0A0A0A` | Page/app background |
| **Surface** | Dark charcoal | `#141414` | Cards, panels, modals |
| **Surface Raised** | Charcoal | `#1E1E1E` | Hover states, elevated surfaces |
| **Border** | Subtle gray | `#2A2A2A` | Thin 1px borders between sections |
| **Text Primary** | Off-white | `#E5E5E5` | Headings, primary content |
| **Text Secondary** | Mid-gray | `#999999` | Labels, timestamps, secondary info |
| **Text Muted** | Dim gray | `#666666` | Placeholders, disabled states |

### Accent

| Role | Color | Hex | Usage |
|------|-------|-----|-------|
| **Primary Accent** | Matrix Green | `#00FF41` | Primary buttons, key actions, active indicators |
| **Accent Muted** | Soft green | `#00FF4126` | Accent backgrounds, subtle highlights (green at ~15% opacity) |

Matrix Green is the signature color. Use it surgically — primary CTAs, active nav states, success indicators, and key data points. Everything else stays in the gray family. When green appears, it should mean something.

### Tier Colors (Semantic)

Each tier has its own color for instant visual recognition:

| Tier | Label | Color | Hex | Meaning |
|------|-------|-------|-----|---------|
| 1 | Bulk / Donate / Trash | Gray | `#666666` | Low value, move on |
| 2 | Quick Flip | Amber | `#F59E0B` | Worth a price tag, sell fast |
| 3 | Research & List | Blue | `#3B82F6` | Needs attention, potential value |
| 4 | High Value / Specialist | Red | `#EF4444` | Stop, secure, and research |

Tier colors appear on badges, status dots, and item cards. They should be immediately scannable — the operator glances at a list and knows what needs attention without reading.

## Typography

**Primary Font:** Plus Jakarta Sans

| Role | Weight | Size Guidance |
|------|--------|---------------|
| **Headings** | 700 (Bold) | Large, clear hierarchy |
| **Body / UI Labels** | 500 (Medium) | Default readable weight |
| **Secondary Text** | 400 (Regular) | Captions, metadata, timestamps |

Keep labels short. "Triage" not "Start Item Triage." "Route" not "Route to Disposition." The UI assumes competence — users are trained, not wandering in off the street.

## Voice & Tone

### Personality

The UI sounds like a sharp coworker, not a corporate manual. It's direct, a little playful, and never wastes your time with obvious instructions.

**Do:**
- Be human and conversational
- Keep microcopy light — a little personality goes a long way
- Use short, punchy labels
- Let whimsy show up in the quiet moments (empty states, loading, transitions)

**Don't:**
- Sound like a SaaS onboarding wizard
- Over-explain things the user already knows
- Use corporate filler ("Please select an option to continue")
- Force humor where urgency or clarity matters more

### Microcopy Examples

| Context | Example |
|---------|---------|
| Empty estate | "No items yet. Time to start digging." |
| AI processing | "Digging through the attic..." |
| Triage complete | "Sorted. 3 worth a closer look." |
| Tier 4 found | "Hold that one. Could be a sleeper." |
| All items resolved | "That's a wrap on this estate." |
| No results | "Nothing here. Maybe check the basement?" |
| Error state | "Something broke. Try that again." |

The playfulness lives in the words, not in the visuals. The UI stays clean and minimal — the copy is where personality shows up.

## UI Principles

1. **Dark canvas, green sparks.** The interface is quiet charcoal. Green draws the eye only where it matters.
2. **Thin borders, not heavy dividers.** 1px borders at `#2A2A2A`. Sections are separated by space and subtle lines, not thick rules or drop shadows.
3. **Density over sprawl.** Operators are processing hundreds of items. Show more, scroll less. Compact layouts with clear hierarchy.
4. **Scannable at a glance.** Tier colors, status dots, and value ranges should be readable from arm's length. No squinting.
5. **Speed over ceremony.** Minimal transitions, no unnecessary modals, no confirmation dialogs for routine actions. The tool gets out of the way.
